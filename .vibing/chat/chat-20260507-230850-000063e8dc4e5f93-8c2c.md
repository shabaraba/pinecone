### import の処理方針

```pinescript
import myIndicator from './myIndicator.pine'
```

これをビルド時に **ファイル内容をインライン展開** する形にします。PineScriptは本来モジュールがないため、`myIndicator.method()` という呼び出しは PineScript v5 の **型メソッド構文** を利用する想定です（`type myIndicator` + `method plot(myIndicator self)`）。

```
dist/ict_killzone.pine  ← そのままコピペ可能
```

### minify の内容
- `//` コメント除去
- 空行・余分な空白除去
- 文字数削減（TradingView の上限対策）

## PineScript v5 ネイティブ `import` の仕組み

### 前提：ライブラリ化が必要

`import` を使うには、対象スクリプトを **`indicator` ではなく `library`** として TradingView に公開（publish）する必要があります。ローカルファイルを直接 import はできません。

```pine
//@version=5
library("MyLib", overlay = true)

export myFunction(float x) =>
    x * 2
```

---

### 呼び出し側：必ず namespace プレフィックスが付く

```pine
//@version=5
import username/MyLib/1 as myLib

// 関数呼び出し
result = myLib.myFunction(close)
```

**インライン展開（名前空間なし）はネイティブでは不可能です。**

---

### 型（UDT）のインポート

```pine
// ライブラリ側（export が必要）
export type MyPoint
    float x
    float y

// 呼び出し側
import username/MyLib/1 as myLib

myLib.MyPoint p = myLib.MyPoint.new(1.0, 2.0)
```

型もメソッドも、すべて `namespace.` プレフィックスが必要です。

---

### メソッドのインポート

ライブラリ側で `method` を定義しても、呼び出し側では **型がライブラリの UDT であるかどうか** によって呼び方が変わります：

```pine
// ライブラリ側
export method plot(MyPoint self) =>
    label.new(bar_index, self.y, "point")

// 呼び出し側
myLib.MyPoint p = myLib.MyPoint.new(1.0, 2.0)
p.plot()  // ← これは動く（型が一致しているため）
```

---
